### 开始构建「曦澪」相关环境
**Anaconda Prompt**
1. **创建虚拟环境（名字就叫 Heliora）：**
   ```bash
conda create -n Heliora python=3.11 -y
   ```
2. **激活这个环境：**
   ```bash

   ```
   *(此时你命令行最左边应该会多出一个 `(heliora)` 的字样，说明你已经进入专属空间了！)*
3. **验证一下：**
   ```bash

   ```

### 第一阶段:经过检查我们电脑是win11精简版，没有虚拟机框架
精简版 Windows”只是被师傅删除了**微软自带的**虚拟机平台（Hyper-V 和 WSL），但这**绝对不影响**你电脑主板 CPU 本身的虚拟化能力（Intel VT-x 或 AMD-V）

#### 安装虚拟机框架（推荐 VirtualBox）
VMware 现在下载注册非常麻烦，推荐你使用开源免费的 **VirtualBox**，它的内核自带驱动，完全不需要 Windows 的底层虚拟机组件支持，完美绕过你的系统报错。
1.  [下载 VirtualBox Windows 版 (v7.0.14)](https://www.google.com/url?sa=E&q=https%3A%2F%2Fdownload.virtualbox.org%2Fvirtualbox%2F7.0.14%2FVirtualBox-7.0.14-161095-Win.exe)

#### 第 2 步：下载一个轻量级的 Linux 系统 (Ubuntu Server)
我们需要一个没有图形界面、只占用极少内存的纯粹 Linux 服务器版，作为我们跑 Docker 的“宿主机”
1. **下载地址：**[清华镜像下载 Ubuntu Server 24.04 LTS (ISO镜像文件)]([https://www.google.com/url?sa=E&q=https%3A%2F%2Freleases.ubuntu.com%2Fjammy%2Fubuntu-22.04.4-live-server-amd64.iso](https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/24.04.4/ubuntu-24.04.4-live-server-amd64.iso))  
    (这个文件大概有3GB，你可以先挂在后台下载。)

### 第二阶段：宿主机 (Windows) 环境预备
系统是精简版 Windows，且采用的是高性能 **AMD Ryzen 7 7840H** 平台，必须完成以下底层打通：
#### 1. 彻底释放虚拟化权限 (关键)
为了让虚拟机能跑在 7840H 的原生速度上，必须关闭 Windows 内核占用：
*   **操作：** 以管理员身份打开 PowerShell，执行：
    ```powershell
    bcdedit /set hypervisorlaunchtype off
    ```
*   **后续：** **必须重启电脑**。
#### 2. 必备工具链安装
*   **Miniconda3：** 管理本地 Python 环境。
*   **Git：** 提供 `Git Bash`（作为 SSH 客户端）和版本控制。
*   **PyCharm Professional：** 核心开发工具，支持远程 SSH 调试。
*   **VirtualBox 7.0+：** 托管曦澪大脑的物理容器

### 第三阶段：曦澪大脑 (Ubuntu) 虚拟化构建

#### 1. 硬件参数
在 VirtualBox 中新建名为 `Heliora-Server` 的虚拟机：
*   **内存：** **8192 MB (8GB)** （你有 32G 内存，给 8G 最稳）。
*   **CPU：** **4 核心** （7840H 动力充足）。
*   **硬盘：** **126 GB** (动态分配)。
*   **引导模式：** **取消勾选 EFI** (为了避开 Ryzen 7000 系在虚拟机里的启动死机报错)。

#### 2. 端口转发
针对校园网+魔法环境，网络采用 **NAT 模式**，并手动设置以下转发：

| 名称          | 主机端口     | 子机端口   | 说明        |
| ----------- | -------- | ------ | --------- |
| **ssh**     | **2026** | **22** | 远程操控大门    |
| **neo4j_h** | 3003     | 7474   | 知识图谱可视化界面 |
| **neo4j_b** | 3036     | 7687   | 图谱数据传输协议  |
| **api**     | 8081     | 8000   | 曦澪后端服务接口  |


#### 3. CPU 伪装补丁 (解决 AMD 启动崩溃)
在 Windows 管理员 PowerShell 执行（让虚拟机认主）：
```powershell
cd "C:\Program Files\Oracle\VirtualBox"
.\VBoxManage.exe modifyvm "Heliora-Server" --cpu-profile "Intel Core i7-6700K"
```

### 第一步： Docker 引擎
在你的这个绿色 SSH 终端里，依次复制并执行下面这两行命令：
1. **更新软件库：**
   ```bash
   sudo apt update
   ```
   *(如果提示输密码，输入 `heliora521`)*
2. **安装 Docker 和 Compose 工具：**
   ```bash
   sudo apt install docker.io docker-compose-v2 -y
   ```
### 第二步：赋予权限（免去以后每次敲 sudo 的烦恼）
安装完后，执行这行命令，把你自己的账号加入 Docker 的管理员组：
```bash
sudo usermod -aG docker $USER
```
### 第三步：刷新权限！
因为刚才给了新权限，当前的终端窗口还没反应过来。请执行最简单的一步：
1. **关掉** PyCharm 下方的这个 Terminal 终端标签页（点那个红色的 X，或者垃圾桶图标）。
2. **重新点开** 一个新的 SSH Session（点那个小箭头 -> `heliora@127.0.0.1:2026`）。
3. 进去后，确认你在项目目录下：
   ```bash
   cd ~/heliora_backend
   ```
4. **指令：**
   ```bash
   docker compose up -d
   ```
### 刷新权限（不用重连）
执行这一行：
```bash
newgrp docker
```
*执行完这行，你的终端可能看起来没变化*

#### 配置 Docker 加速镜像
我们要给 Docker 指定几个目前国内还能用的“传送门”（比如南京大学、DaoCloud 等）：
```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.nju.edu.cn",
    "https://docker.m.daocloud.io",
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ]
}
EOF
```
#### 重启 Docker 引擎使配置生效
```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```
#### 再次启动
现在网络通道已经重定向到国内镜像了，再次执行：
```bash
docker compose up -d
```
等待下载完成


#### 下载完成后在 PyCharm 的绿色终端输入：
```bash
docker compose ps
```
