# 术语统一

| 统一术语 | 英文/缩写 | 类型 | 核心含义 | 对应字段/对象 |
|---|---|---|---|---|
| 长期记忆服务 | Memory Service | 模块 | 负责跨会话记忆写入、检索、冲突治理、生命周期管理 | memory_records, memory_events |
| 证据绑定 | evidence binding | 规则 | 每条记忆必须绑定来源对话证据，禁止无证据写入 | evidence, source_turn_ids, trace_id |
| 写入门控 | Memory Gate | 策略 | 判断是否值得写入长期记忆 | S_write, tau_write(d) |
| 候选激活 | candidate_active | 状态 | 通过门控并进入巩固队列的候选记忆 | status=candidate_active |
| 候选保留 | candidate_hold | 状态 | 分数接近阈值，先保留等待后续证据 | status=candidate_hold |
| 冲突态 | conflicted | 状态 | 新旧证据不充分决断，需要澄清或人工审核 | status=conflicted |
| 过时态 | deprecated | 状态 | 被新证据替代的旧记忆，不物理删除 | status=deprecated |
| 归档态 | archived | 状态 | 强度衰减后归档，默认不自动硬删除 | status=archived |
| 软删除 | soft delete | 生命周期动作 | 逻辑删除并进入索引清除流程 | status=deleted + 异步清除 |
| 安全策略模式 | security policy mode | 治理开关 | 控制策略执行级别，支持 strict 与 trusted_local_max | SECURITY_POLICY_MODE |
| 冲突判别值 | Lambda | 算法量 | 新旧记忆后验比的对数判别值 | lambda_value |
| 图谱扩展召回 | graph expansion recall | 检索阶段 | 通过图关系做多跳候选扩展，属于P1标准组件 | Neo4j + PPR |
| 审计事件 | memory_events | 治理对象 | 记录写入、冲突、回滚、删除等关键动作 | memory_events 表 |
