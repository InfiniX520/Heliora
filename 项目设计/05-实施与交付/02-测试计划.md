# Heliora(曦澪) 当前测试清单

测试目标只有一个：保证主流程能跑通，失败时能定位。

---

## 1. 当前必测范围

只测 7 条主流程：
1. 插件发送错误上下文。
2. 后端接收并保存错误记录。
3. 系统返回引导式定位与下一步建议。
4. 生成一次答辩演练反馈。
5. 文件整理执行后可回滚。
6. 记忆写入门控与检索命中。
7. 记忆冲突处理与删除SLA闭环。

---

## 2. 每条流程的通过标准

### 2.0 记忆专项量化门槛（与最终方案对齐）

除流程跑通外，记忆系统需满足以下硬门槛：
1. Recall@K >= 0.85。
2. Precision@K >= 0.80。
3. Conflict Resolution Accuracy >= 0.90。
4. Hallucinated Memory Rate <= 2%。
5. 对话首字延迟增量 P95 <= 120ms。
6. 删除请求闭环SLA <= 5分钟。
7. 单条冲突未决时长 <= 24小时。

说明：
1. P0 阶段可先做小样本验证，但必须保留指标采集与报表出口。
2. P1 起必须覆盖图谱扩展召回路径（Neo4j + PPR）验证。

### 用例1：插件发送错误
1. 输入：错误信息 + 文件路径 + 代码片段。
2. 通过：后端返回成功状态与 task_id。

### 用例2：错误记录落库
1. 输入：来自用例1的请求。
2. 通过：数据库存在对应记录，字段完整。

### 用例3：引导式纠错
1. 输入：真实报错样例。
2. 通过：返回结构包含结论、依据、下一步。

### 用例4：答辩演练
1. 输入：项目上下文。
2. 通过：生成 10 题与结构化反馈。

### 用例5：文件管家回滚
1. 输入：一个测试目录。
2. 通过：执行整理后可一键恢复到原状态。

### 用例6：长期记忆写入与检索
1. 输入：包含明确偏好的多轮对话。
2. 通过：产生含 evidence 的记忆记录，并在后续对话命中。

### 用例7：冲突与删除闭环
1. 输入：同一偏好给出相反新证据 + 删除请求。
2. 通过：冲突进入五区间决策流程；删除请求5分钟内完成软删与索引清除。

---

## 3. 每次开发后的固定动作

1. 跑完 7 条主流程。
2. 记录失败用例与日志位置。
3. 修复后重跑失败用例。
4. 全部通过后再进入下一任务。

---

## 3.1 CI 门禁（新增）

1. RabbitMQ E2E 已纳入 CI 强制门禁，路径：`.github/workflows/backend-rabbitmq-gate.yml`。
2. 门禁必须通过 `tests/test_tasks_rabbitmq_e2e.py`，不允许以 skip 通过。
3. CI 环境使用 `RABBITMQ_E2E_REQUIRED=true`；若 RabbitMQ 不可用，测试直接失败。
4. CI 增加 Ruff 静态检查门禁：`python -m ruff check app tests`。
5. CI 增加 Mypy 类型检查门禁：`python -m mypy app`。
6. CI 增加覆盖率门禁：`python -m pytest --cov=app --cov-report=term-missing --cov-fail-under=70 -q`。
7. 任一门禁失败即阻断合并。
8. CI 新增 PostgreSQL 冒烟门禁：`TASK_PERSISTENCE_BACKEND=postgres` + `TASK_QUEUE_BACKEND=memory`，执行 `001_up -> tests/test_tasks_submit.py -> 001_down`。
9. CI 新增 RabbitMQ + PostgreSQL 组合门禁：`TASK_PERSISTENCE_BACKEND=postgres` + `TASK_QUEUE_BACKEND=rabbitmq`，执行 `001_up -> tests/test_tasks_rabbitmq_e2e.py -> 001_down`。
10. RabbitMQ E2E 用例已补充持久层断言，强制校验 `task_state_store/task_event_store` 写入成功，避免仅内存路径通过。
11. CI 新增环境一致性门禁：`python scripts/validate_env_consistency.py --env-file .env`，用于阻断 DSN/账号/密码漂移。

## 3.11 本地部署与云端测试职责边界

通过标准：
1. 明确本地部署不等于取消云端测试；两者职责不同且互补。
2. 发布前必须同时具备：
	- 本地/VM 预检与演练通过（运行可行性）
	- 云端 CI run 与 artifact 证据齐全（质量可审计）
3. 若仅满足其一，不进入最终 Go 签字。

执行口径：
1. 本地环境负责“运行结果真实性”（真实配置、真实依赖、真实切流动作）。
2. 云端 CI 负责“验证一致性与可复核性”（统一执行器、标准门禁、链接可追溯）。
