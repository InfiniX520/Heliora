## 1. IDE 使用策略（你已具备 IDEA + PyCharm）

结论:
1. 后端主开发使用 PyCharm（Python、调试、测试、数据库联调效率最高）。
2. 前端与桌面壳、文档协作可使用 IDEA。
3. 若只开一个 IDE，优先开 PyCharm。

建议分工:
1. PyCharm:
   - Orchestrator / Memory Service / TaskDispatcher / API / 测试。
2. IDEA:
   - Vue/Electron 前端、接口联调脚本、文档整理。

## 2. 开发环境基线（Windows）

### 2.1 必装软件

1. Git（最新稳定版）。
2. Python 3.11+（优先 3.11；Ubuntu 24.04 使用 3.12 也可先开发）。
3. Node.js 22 LTS。
4. Docker Desktop（启用 WSL2）。
5. PyCharm Professional / IDEA Ultimate。

### 2.2 Python 版本策略（处理“现在变成 3.12”）

1. 当前 `python3 --version = 3.12.x` 属于 Ubuntu 24.04 默认行为。
2. 当前项目依赖可在 3.12 运行，先不阻塞开发。
3. 生产基线建议 3.11，稳定阶段再统一切换。
4. 无论 3.11/3.12，都必须使用项目虚拟环境，不使用系统 `pip3` 安装项目依赖。
5. Day-0 推荐命令: `bash scripts/bootstrap_dev.sh`。

### 2.3 常见报错速查

1. `python: command not found`
   - 原因: 未激活虚拟环境，且系统默认只有 `python3`。
   - 处理: `python3 -m venv .venv && source .venv/bin/activate`。
2. `pytest: command not found`
   - 原因: 虚拟环境依赖未安装。
   - 处理: `pip install -r requirements.txt` 后再执行 `pytest`。
3. `/health: No such file or directory`
   - 原因: 把 HTTP 路径当成 shell 命令。
   - 处理: 先运行 `python main.py`，再执行 `curl http://127.0.0.1:8000/health`。

### 2.4 后端依赖服务

1. PostgreSQL 15+
2. Redis 7+
3. Neo4j 5+
4. RabbitMQ 3.13+
5. MinIO

说明:
1. 本地开发建议使用 Docker 统一管理服务版本，避免环境漂移。

## 3. 配置开关基线（必须先定）

关键配置以 [04-配置管理规范](../../04-技术规范/04-配置管理规范.md) 为准。

最低要求:
1. `ENABLE_MEMORY_SERVICE=true`
2. `MEMORY_GRAPH_RETRIEVAL_P1=true`
3. `SECURITY_POLICY_MODE=strict`（默认）
4. `LOCAL_MAX_PRIVILEGE_ACK=false`（默认）

本地高权限模式（可选）:
1. 仅本地调试时开启 `SECURITY_POLICY_MODE=trusted_local_max`。
2. 同时设置 `LOCAL_MAX_PRIVILEGE_ACK=true`。
3. 必须保留确认、审计、回滚链路，不可跳过。
