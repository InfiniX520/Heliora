# 发布窗口切流执行单（Task Persistence SQLite -> PostgreSQL）

> 版本: v1.0
> 日期: 2026-04-05
> 适用范围: heliora_backend 任务持久化链路

---

## 1. 目标

在不影响现网可用性的前提下，完成任务持久层从 sqlite 到 postgres 的可控切换，并保留一键回滚能力。

---

## 2. 前置条件（必须全部满足）

1. 云端 CI `backend-rabbitmq-gate.yml` 全部门禁通过。
2. 发布前一键预检通过（推荐作为主入口）：
   - bash scripts/preflight_release_window_vm.sh
3. VM 一键矩阵回归通过：
   - bash scripts/run_task_persistence_pg_matrix_vm.sh
4. VM 切换+回滚演练通过：
   - bash scripts/rehearse_task_persistence_cutover_rollback_vm.sh
5. `.env` 一致性校验通过：
   - python scripts/validate_env_consistency.py --env-file .env
6. 已确认本次窗口有明确回滚负责人和值班联系人。

---

## 3. 发布窗口时间线（建议）

1. T-60 分钟: 冻结后端变更、确认依赖容器健康。
2. T-45 分钟: 执行一次云端 CI，保存门禁摘要与 artifact 链接。
3. T-30 分钟: 执行 VM 切换+回滚演练，保存命令输出。
4. T-15 分钟: 评审 Go/No-Go。
5. T0: 切换 `TASK_PERSISTENCE_BACKEND=postgres` 并重启后端。
6. T+15 分钟: 完成首轮线上 smoke（submit/status/events/cancel）。
7. T+60 分钟: 观察窗口结束，确认是否关闭回滚待命。

---

## 4. 切流执行步骤

### 4.1 切流前最终确认

1. 推荐执行一键预检（自动生成报告）：
   - bash scripts/preflight_release_window_vm.sh
2. 预检报告位置（默认）：
   - .release-reports/release_preflight_YYYYMMDD_HHMMSS.md
3. 校验配置一致性：
   - python scripts/validate_env_consistency.py --env-file .env
4. 校验基础依赖：
   - bash scripts/check_services.sh
5. 执行切换+回滚演练：
   - bash scripts/rehearse_task_persistence_cutover_rollback_vm.sh

### 4.2 正式切流

1. 修改 `.env`:
   - TASK_PERSISTENCE_BACKEND=postgres
2. 保持 DSN 显式一致（推荐）：
   - TASK_REGISTRY_POSTGRES_DSN 与 DATABASE_URL 一致
   - TASK_EVENTS_POSTGRES_DSN 与 DATABASE_URL 一致
3. 重启 API/worker：
   - bash scripts/stop_api.sh
   - bash scripts/start_api_bg.sh

### 4.3 切流后验证

1. 接口 smoke:
   - 提交任务（submit）
   - 查询状态（status）
   - 查询事件（events）
   - 取消任务（cancel）
2. 队列链路 smoke:
   - memory 路径验证
   - rabbitmq 路径验证
3. 观察日志：
   - 不允许出现 task state/event persistence failed 关键字

---

## 5. Go / No-Go 判定

满足以下全部条件才允许继续保持 postgres:

1. CI 全部门禁通过。
2. 切换后 smoke 全通过。
3. 关键错误日志为 0（持久化失败、外键失败、连接认证失败）。
4. 15 分钟观察窗口内无连续告警。

任一条件不满足，立即执行回滚。

---

## 6. 回滚触发条件

任一条件满足即触发回滚：

1. submit/status/events 任一核心接口失败率显著升高。
2. 持久化写入错误持续出现（>= 3 次/5 分钟）。
3. PostgreSQL 连接/认证错误无法在 5 分钟内恢复。
4. RabbitMQ + PostgreSQL 组合链路不可用。

---

## 7. 回滚步骤（sqlite）

1. 修改 `.env`:
   - TASK_PERSISTENCE_BACKEND=sqlite
2. 重启 API/worker：
   - bash scripts/stop_api.sh
   - bash scripts/start_api_bg.sh
3. 执行 sqlite 回归：
   - python -m pytest tests/test_tasks_submit.py -q -rA
4. 记录回滚时间、触发原因、恢复时间与影响范围。

## 10. 本地部署与云端测试边界说明（FAQ）

1. 本项目是本地化部署，运行位置在本机/自有 VM；这不等于放弃云端 CI。
2. 云端 CI 的职责是“统一门禁 + 跨环境回归 + 可审计证据沉淀”，不是“把系统部署到云上”。
3. 发布判定采用三段式证据：本地预检报告 + 云端门禁 run/artifact + 切流后 smoke 结果。
4. 若仅有本地通过而无云端证据，结论只能是阶段通过，不进入最终发布签字。
5. 本批次完整证据索引见：`07-发布证据索引与检索指南.md`。
